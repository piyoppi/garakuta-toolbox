ブログを https://garakuta-toolbox.hatenablog.com からお引越してみます。
しばらくはこちらのブログ環境を使ってみて、様子を見ながら手を入れたりしようと思っています。

（あくまでお試しということで、場合によっては元のブログ環境に戻ったりするかもしれません）

## なんでお引越ししたの？

今までのブログ環境にも概ね不満はなかったのですが、以下のような理由でお引越しすることにしました。

### ソースコードをよいかんじに見せたい

今までの環境でもソースコードを掲載することはできましたが、一部の言語でシンタックスハイライトが思い通りにならずにもやもやを感じていました。

新環境では（私が）満足するような形でソースコードを掲載できるようになりました。

```html
<html>
  <head></head>
  <body></body>
</html>
```

```javascript
const hoge = 'fuga'
console.log(hoge)
```

### 好きなテキストエディタで編集したい

一般的なブログサービスでは、基本的にはWebブラウザから記事を編集することになるかと思います。

ブログ記事の執筆のために既存のソースコードをブログ記事にコピペしようとすると、テキストエディタとブラウザをいったり来たりしなくてはなりませんし、ブログ記事にペーストした後にインデントを調整したりなどの修正が発生しがちでした。

ブログ記事を書くエディタとソースコードを書くエディタが同じだと、コピー＆ペーストの点で都合がよいと思っています。
（普段の開発にはWindows + WSL + Windows Terminal + Vim という環境でコーディングをしていますが、記事の執筆にも使い慣れたVimを使いたいという事情もあります。）

### ブログテンプレートを自由にいろいろしたい

ある程度デザインを自由にできるといいなと思っていました。また、自作コンポーネントの実験をブログ記事上で行うなどもしたいなと思っていましたので、デザイン以外にもある程度カスタマイズできるとよいなという気持ちがありました。

## このブログ環境はどうやってつくったの？

[Gatsby](https://www.gatsbyjs.com/) を使って作りました。いままでわたしのポートフォリオサイト的な役割をしていた https://garakuta-toolbox.com は Gatsbyをベースに作っていましたが、ここにブログ機能を足した形になっています。
Gatsbyをちゃんと使っておきたいということも乗り換え理由の一つです。

ほぼ [チュートリアル](https://www.gatsbyjs.com/docs/tutorial/) に倣って実装しています。
